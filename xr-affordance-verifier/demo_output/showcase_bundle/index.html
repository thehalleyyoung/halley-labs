What changed

Use this list as the voice-over for the live demo. Each item corresponds to a deliberate product or layout correction.

  • Raised the floor-level and ceiling-level controls into an ergonomic reach envelope.
  • Separated clustered controls and enlarged the tiny activation target for comfortable selection.
  • Moved the out-of-bounds affordance back into the main workspace volume.
  • Restored missing labels and feedback channels so the UI is understandable and responsive.
  • Fixed the two-handed affordance to require `BothHands` and richer feedback, matching the intended interaction semantics.

Broken-scene lint guidance

The original scene already contains repair hints. Use them to justify why the fixes below are not arbitrary.

  • too_low_btn: Move element to at least 0.40m height
  • too_high_btn: Move element to at most 2.20m height
  • tiny_btn: Increase the activation volume size
  • no_label_btn: Add a descriptive label for screen readers
  • no_feedback_btn: Add visual, haptic, or audio feedback
  • bad_twohanded: Element 'bad_twohanded' is TwoHanded but actuator is not BothHands
  • close_a: Increase spacing to at least 0.030m
  • bad_twohanded: Element 'bad_twohanded' requires TwoHanded interaction not supported by any configured device
