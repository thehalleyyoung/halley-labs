# Review: Certified CSL Model Checking via Tensor-Train Decomposition of Chemical Master Equation States

**Reviewer:** Sara Roy (ML + Formal Verification)
**Score: 7/10**

---

TN-Check sits at a fascinating intersection of numerical methods and formal verification, and the ML-verification perspective reveals both opportunities and gaps. The most ML-relevant component is the adaptive rank controller: the proposal describes a greedy-doubling heuristic with per-bond singular-value monitoring, deferring a learned MLP rank controller to follow-up work unless ablation shows >3× improvement. This is a reasonable scoping decision, but the greedy-doubling heuristic is a missed opportunity for demonstrating ML-verification integration. The rank adaptation problem is well-suited to reinforcement learning: the state is the current bond-dimension profile and singular-value spectrum at each bond, the action is a bond-dimension adjustment (increase/decrease at each bond), and the reward is verification accuracy per unit compute. A simple Q-learning or contextual bandit approach could learn bond-dimension allocation policies that outperform uniform-rank or greedy-doubling strategies, especially for models with heterogeneous entanglement structure (e.g., a signaling cascade where upstream species are weakly correlated but downstream effector species are strongly correlated). The decision to defer this to follow-up work is understandable given the project's scope, but a pilot experiment on 2–3 models would validate whether learned rank control is worth pursuing.

The neural network verification connection is indirect but worth highlighting. TN-Check verifies stochastic reaction networks, not neural networks, but the mathematical infrastructure—tensor decomposition, error-bounded numerical computation, temporal-logic specification—could potentially be adapted for verifying stochastic neural networks or Bayesian neural networks, where the posterior distribution over weights is the analog of the CME probability distribution over species copy numbers. This is speculative but represents a research direction that could emerge from the TN-Check framework. More concretely, the error certification methodology—Theorem 1's error-contraction analysis with explicit dependence on truncation tolerance, time horizon, and number of truncation steps—is a model for how ML verification systems should handle approximate computation. Current neural network verification tools (α,β-CROWN, VeriNet, MN-BaB) provide sound but often loose bounds because they lack principled error composition across layers. TN-Check's approach of tracking per-bond truncation error and composing it through the Markov semigroup is analogous to tracking per-layer approximation error through a neural network's computation graph.

The training rigor concern applies to the deferred MLP rank controller: if implemented, it must be trained on a diverse set of reaction networks to avoid overfitting to specific model topologies. The BioModels repository provides sufficient diversity (~1000 curated models), but the training data generation pipeline—running TT computations at multiple rank settings on each model and recording accuracy-vs-cost tradeoffs—is computationally expensive. The proposal should specify the computational budget for rank-controller training and the held-out test set for evaluating generalization. The three-valued semantics for nested CSL operators is a strong ML-verification contribution: rather than providing a binary pass/fail result with uncertain accuracy (as current neural network verifiers often do), TN-Check explicitly quantifies its uncertainty and reports "indeterminate" when the error bound is too large. This honest uncertainty quantification is a design pattern that ML verification tools should adopt. The evaluation plan's statistical MC comparison—TN-Check vs. Gillespie SSA with 10⁶ trajectories—is particularly valuable for rare-event properties (probability < 10⁻³), where the TT-based approach provides certified bounds that SSA cannot match without prohibitively many samples.
