# Review: Certified CSL Model Checking via Tensor-Train Decomposition of Chemical Master Equation States

**Reviewer:** Joseph S. Chang (Automated Reasoning & Logic Foundations)
**Score: 8/10**

---

The logical foundations of TN-Check are well-structured, and the proposal demonstrates clear understanding of the proof-theoretic requirements for certified verification. The CSL semantics follow Baier and Katoen's standard treatment, which is well-established and not a source of logical concern. The novel logical content lies in the interaction between CSL's semantic operators and TT compression, particularly the three-valued semantics for nested probability operators. The interval-arithmetic approach—classifying states with probability within ε of the threshold as "indeterminate"—is logically sound: it weakens the classical two-valued CSL semantics to a three-valued logic where the truth value of P≥p[φ] at state s is {true} if Prob(s ⊨ φ) ≥ p + ε, {false} if Prob(s ⊨ φ) < p − ε, and {indeterminate} otherwise. This three-valued logic preserves soundness (no false positives or false negatives for states classified as true/false) at the cost of completeness (indeterminate states receive no verdict). The key proof obligation is that the ε used in the three-valued classification correctly bounds the TT truncation error at each nesting level—this is where Theorem 1's error-contraction analysis is load-bearing.

The proof-theoretic structure of the four load-bearing theorems deserves careful assessment. Theorem 1 (Error Contraction) is the soundness foundation: if the error bound C(t)·ε_round is incorrect, all certified results are invalid. The proof strategy exploiting the Metzler structure of Q is mathematically sound—the ℓ¹ contractivity of the Markov semigroup is a classical result (e.g., Norris, "Markov Chains," 1997)—but the extension to TT-compressed computation with clamping is non-trivial. The clamping operation (setting negative entries to zero and renormalizing) is a projection onto the probability simplex, and the contraction of this projection in ℓ¹ norm is not immediate. The proof must show that clamping does not increase the ℓ¹ distance from the exact solution, which requires a monotonicity argument: the exact solution p_exact is non-negative, so clamping the approximate solution p_SVD (which may have negative entries) can only move it closer to p_exact in the ℓ¹ sense. This argument is plausible but requires careful verification, especially when the SVD approximation has large negative entries (which can occur when the probability distribution has sharp peaks). Theorem 3 (Complexity Separation) is the most ambitious from a proof-theoretic standpoint, requiring a perturbative expansion of the Markov semigroup combined with TT rank bounds. The key step is bounding the TT rank of e^{(Q_local + δ·Q_coupling)t} using the Baker-Campbell-Hausdorff expansion or a Trotter-Suzuki decomposition. The rank bound poly(m)·(1+δt)^O(k) suggests a polynomial approximation of the matrix exponential with rank growth controlled by the coupling strength—this is consistent with known TT rank bounds for smooth functions (Khoromskij 2011), but the extension to non-Hermitian generators requires additional care.

The decidability and complexity landscape of CSL model checking on TT-compressed representations introduces new questions for the automated reasoning community. Classical CSL model checking on explicit-state CTMCs is polynomial in the state space size (Baier et al., 2003). TN-Check replaces the explicit state space with a TT-compressed representation, making the effective complexity depend on bond dimension χ rather than state space size M^N. The key question is whether CSL model checking on TT-compressed representations is polynomial in χ—and the answer appears to be yes for single-level properties (where each operation preserves or moderately inflates bond dimension) but potentially exponential for deeply nested properties (where error accumulation may force bond dimension growth). The trusted computing base analysis is implicit but important: the user trusts the Rust TT arithmetic engine (~15K LoC), the CME-to-MPO compiler (~18K LoC), the CSL checker (~20K LoC), and the error certification monitor (~8K LoC)—a total TCB of ~60K LoC, which is large by formal verification standards. The error certification monitor is itself part of the TCB: if it underestimates error, the certified bounds are wrong. A formal verification of the error monitor's logic in a proof assistant would significantly strengthen the soundness claim, though this is admittedly a substantial additional effort.
