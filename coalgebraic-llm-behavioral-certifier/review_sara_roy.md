# Review: CABER — Coalgebraic Behavioral Auditing of Foundation Models

**Reviewer:** Sara Roy
**Persona:** Machine Learning & Formal Verification
**Expertise:** ML evaluation methodology, formal verification of ML systems, statistical rigor, experimental design, ablation studies, reproducibility

---

## Summary

CABER presents an ML-and-formal-methods hybrid system for black-box LLM behavioral auditing, combining PCL* automata learning, QCTL_F temporal logic model checking, CoalCEGAR abstraction refinement, and Kantorovich bisimulation distance. The experimental methodology is substantially more rigorous than most papers in this space—the transparent limitation disclosure, comprehensive statistical baselines, and systematic ablation studies set a standard that other LLM analysis papers should follow. However, the evaluation suffers from critical empirical limitations: a single non-frontier model (gpt-4.1-nano), classifiers that do not generalize (0.55–0.69 LOPO accuracy), poorly calibrated graded scores (0.28–0.73 error), insufficient experimental scale (270 total API calls), and a fundamental gap between the theoretical guarantees and what the empirical evidence actually demonstrates.

## Strengths

1. **Exemplary methodological honesty.** The paper is unusually transparent about its limitations: PAC bound vacuity at operating sample sizes, classifier non-generalization on unseen prompts, non-functorial abstraction with 0–27% inconsistency, CoalCEGAR monotonicity violation for 1/5 configurations, PDFA baseline weakness, and the corrected query complexity (O(β·n₀) → O(2^β·n₀)). This level of self-critical reporting is rare in the LLM safety literature and should be recognized as exemplary scientific practice. The paper explicitly states it provides "approximate behavioral testing, not formal verification"—a crucial distinction most comparable papers fail to make.

2. **Comprehensive baseline battery with proper statistical methodology.** Comparing against six baselines—chi-squared tests, MMD with RBF kernel, KL divergence estimation, token-frequency counting, random sampling, and simplified ALERGIA-based PDFAs—with consistent evaluation metrics and statistical significance testing provides thorough contextualization. The 7/15 divergent prompts finding is corroborated by multiple independent methods, establishing the core empirical result's robustness.

3. **Systematic ablation coverage.** Ablations over distance metrics (total variation, Jensen-Shannon, Hellinger), PCL* hyperparameters (alphabet size, tolerance, maximum states), classifier variants (keyword, embedding, hybrid), alphabet sensitivity analysis, and CoalCEGAR refinement levels provide a thorough dissection of the system's design choices. This level of ablation is essential for understanding which components contribute to performance and which are incidental.

4. **Bayesian uncertainty quantification over frequentist defaults.** Using Beta-Binomial posteriors with highest-posterior-density intervals for divergence rate estimation is a sound statistical methodology given the small sample sizes. The posterior P(divergence rate > 25%) = 97.3% with 95% HPD [25.0%, 69.9%] is more informative than a p-value, especially when sample sizes preclude asymptotic frequentist guarantees.

5. **Substantial codebase with property-based testing.** The ~99K lines of Rust and Python implementation with 38 property-based tests validating mathematical invariants demonstrates serious engineering commitment. The property-based tests (functor naturality, bisimulation metric space properties, model-checker convergence) serve as both correctness checks and documentation of the mathematical structure.

## Weaknesses

1. **Single model, single tier, zero external validity.** All 270 API calls target gpt-4.1-nano, a non-frontier model. No experiments on GPT-4, GPT-4o, Claude 3.5 Sonnet, Gemini 1.5 Pro, Llama 3, or any other model. No cross-tier validation (nano vs. standard vs. frontier). No open-source model experiments where behavioral distributions can be verified against known internal structure. The external validity of every finding is essentially zero: the 7/15 divergent prompts, the ≤40-state automata, the ≥92% prediction accuracy, the Kantorovich distances—all of these are specific to a single model at a single API snapshot in time. For an evaluation framework, this is the most critical limitation. Theorem 1's convergence guarantee and Theorem 2's bandwidth-sample complexity bound are model-agnostic in theory but untested across models in practice.

2. **Classifier does not generalize—the "model" in model checking is not a model.** Leave-one-prompt-out cross-validation yields 0.55–0.69 accuracy—barely above chance for a 4-class problem (random baseline: 0.25). This means the behavioral automaton learned from 15 queried prompts has no predictive power on a 16th unseen prompt. The learned automaton is not a model of the LLM's behavioral space; it is a lookup table for the specific prompts queried. This invalidates the primary value proposition of model checking: model checking is valuable because verified properties hold for all states reachable from the initial state, but if the model only represents observed states and cannot predict unobserved states, the verification guarantee covers only a negligible fraction of the behavioral space. The ≥92% prediction accuracy is on the training set; the generalization accuracy of 0.55–0.69 is the operationally relevant metric.

3. **Calibration error undermines the graded semantics differentiator.** Calibration error of 0.28–0.73 for graded satisfaction scores means the continuous [0,1] values are unreliable probability estimates. Since graded satisfaction is CABER's primary differentiator from binary testing frameworks, this miscalibration strikes at the core contribution. At 0.73 calibration error, a satisfaction degree of 0.8 could correspond to a true probability anywhere from near 0 to near 1. No calibration curves are presented, no reliability diagrams, and no post-hoc calibration methods (Platt scaling, temperature scaling, isotonic regression) are applied. For a framework grounded in graded logic, this is an unacceptable gap.

4. **Insufficient experimental scale undermines statistical conclusions.** The total of 270 API calls (3 configurations × 15 prompts × ~6 trials) is extremely small. The behavioral space of a language model is effectively infinite; 15 prompts cannot characterize it. No power analysis justifies that 5–6 trials per prompt suffice for reliable behavioral estimation—given the stochastic nature of LLM responses, the variance of behavioral atom assignments across trials is likely high relative to the between-configuration signal. The Bayesian posterior HPD interval of [25.0%, 69.9%] for the divergence rate illustrates the wide uncertainty: the framework cannot determine whether 25% or 70% of prompts exhibit divergent behavior. More prompts and more trials would narrow this interval substantially.

5. **Statistical baselines match detection power—structural advantages are undemonstrated.** Chi-squared and MMD tests detect the same behavioral divergences at comparable significance levels. The paper argues qualitatively that coalgebraic methods provide structural advantages (reusable automata, temporal reasoning, compositional specifications), but provides no quantitative evidence that these advantages yield: (a) detection of divergences that statistical baselines miss, (b) faster detection with fewer queries, (c) more actionable diagnostic information, or (d) better downstream decision-making. Without such evidence, a reviewer must ask: given that chi-squared tests require ~10 lines of Python and CABER requires ~99K lines of Rust/Python, what is the marginal value?

6. **No end-to-end evaluation of practical utility.** There is no downstream task demonstrating that CABER's behavioral audits lead to better model selection, safer deployment decisions, more effective red-teaming, or improved model debugging. The framework produces automata, satisfaction degrees, and bisimulation distances—but no user study, no deployment case study, and no comparison of decision quality (with CABER vs. without) is presented. For a tool paper with ~99K lines of code, the absence of any practical utility evaluation is a significant gap. The structural outputs (automata, temporal logic verdicts) are interesting but their practical value is assumed, not demonstrated.

## Questions for Authors

- What sample size (prompts × trials) would you need for the classifier to achieve >80% leave-one-prompt-out accuracy, and is this feasible at reasonable API cost? Is the non-generalization fundamental (LLM behavioral spaces are too complex for finite automata) or addressable with more data?
- Have you tested on any other model—GPT-4, Claude, Gemini, or even an open-source model like Llama 3 where ground-truth behavioral distributions could be verified? What do you expect would change?
- Can you demonstrate a concrete scenario where CABER's structural output (learned automaton, temporal logic verdict, bisimulation distance) leads to an actionable insight that a simple chi-squared test on behavioral atom frequencies would miss?

## Overall Assessment

CABER is methodologically more rigorous than virtually any comparable LLM behavioral analysis paper. The transparent limitation disclosure, comprehensive baselines, and systematic ablations set a standard that other LLM safety papers should aspire to. The coalgebraic theoretical framework is genuinely sophisticated and identifies the right formal structure for behavioral auditing. However, the empirical evaluation does not demonstrate that this sophisticated structure produces better results than simple alternatives: the single-model scope, non-generalizing classifier, miscalibrated graded scores, small experimental scale, and comparable performance of trivial statistical baselines collectively mean the theoretical contributions remain promissory rather than empirically validated. The work is valuable as a research agenda that maps out the formal structure needed for LLM behavioral verification, but the current empirical results do not yet justify the coalgebraic complexity over simpler approaches. The honest limitation reporting is the paper's greatest empirical strength—paradoxically, the limitations it documents are also the strongest arguments against its current practical value.

**Score:** 4/10
**Confidence:** 5/5
