Community: the software engineering/formal methods research community

Idea: Incremental Refinement Type Inference for Dynamic Languages: Automatically infers refinement types (types decorated with logical predicates, e.g., {x: int | x > 0 && x < len(arr)}) for Python and TypeScript codebases by combining abstract interpretation over the program's concrete semantics with SMT-backed predicate inference (Liquid Types style). Given a codebase, produces per-function contracts that are machine-checkable: array bounds safety, null/undefined safety, division-by-zero freedom, and type-tag correctness. Uses counterexample-guided predicate abstraction refinement (CEGAR) when initial predicates are insufficient. Crucially incremental: re-analyzes only changed functions and their transitive dependents on each commit, making it practical for CI pipelines. ~165K LoC spanning: Python type-aware AST→SSA compiler (25K), TypeScript AST→SSA compiler (25K), abstract domain library (numeric, pointer, string, array shape) (30K), predicate inference and CEGAR loop (25K), SMT encoding and Z3 interface (15K), incremental dependency tracker and invalidation engine (20K), counterexample visualization and contract output (10K), test suite and benchmark infrastructure (15K).
