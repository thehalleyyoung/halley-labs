# Decidable Refinement Subtyping over Mutable Heap Abstractions with Incremental Fixed-Point Maintenance

## Problem and Approach

Liquid Types and their descendants solve refinement type inference for ML-family languages by reducing subtyping checks to SMT validity queries over a fixed, finite set of candidate predicates. This works because ML programs inhabit a world of immutable bindings and algebraic data — the refinement logic lives in a decidable fragment (typically QF_LIA or QF_UFLIA), and the type structure is rigid enough that Hindley-Milner-style constraint generation produces a finite system of Horn clauses. Dynamic languages like Python and TypeScript violate every assumption in this pipeline simultaneously: (1) variables are mutable and untyped, so refinements must track heap-dependent predicates whose validity changes at every program point; (2) objects are open dictionaries with runtime-determined keys, so the "shape" of a type is itself a predicate over an unbounded domain; (3) higher-order functions, decorators, and metaprogramming create calling contexts that cannot be enumerated syntactically. The mathematical question is whether refinement subtyping remains decidable — and practically solvable — when the domain includes mutable heap abstractions, structural object types with width subtyping, and predicate families quantified over dynamic keys.

We propose a new type-theoretic framework: **heap-indexed refinement types** over a separation-logic--inspired abstract domain. Each function signature is decorated with a refinement that speaks not only about the value, but about a footprint of the heap it may read or write, using a fragment of separation logic restricted to the *points-to* and *list-segment* predicates. Subtyping becomes an entailment check in a combined theory: linear arithmetic + uninterpreted functions + a *bounded heap fragment* we define. The central theoretical contribution is a proof that this combined theory, which we call **QF_UFLIA+H**, admits quantifier-free interpolation, which is the key property needed for CEGAR-based predicate discovery to converge. Without interpolation, CEGAR loops forever or produces predicates outside the decidable fragment — the system is useless in practice.

The incrementality requirement introduces a second mathematical challenge: when a function body changes, we must re-compute only the portion of the global type-inference fixed point that is affected. We model the whole-program analysis as a system of recursive Horn clauses over a product lattice (predicate abstraction × heap abstraction × type-tag abstraction), and we develop an **incremental Datalog-style fixed-point maintenance algorithm** that propagates delta facts through the clause graph. The theoretical question is whether this delta propagation is *sound and complete* — i.e., whether the incrementally maintained solution equals the from-scratch solution — under the non-monotone negation that arises from refinement type well-formedness checks (e.g., "this predicate does NOT hold at the call site, so the callee's precondition is violated"). We prove soundness and completeness by exhibiting a stratification of the clause system that is preserved under incremental updates.

## New Mathematics Required

### Theorem 1: Quantifier-Free Interpolation for QF_UFLIA+H
**Statement.** Let QF_UFLIA+H be the quantifier-free fragment of the theory combining linear integer arithmetic, uninterpreted functions, and a bounded heap theory H (points-to assertions `x ↦ v`, separating conjunction `*` restricted to acyclic footprints of bounded depth *k*). If φ ∧ ψ is unsatisfiable in QF_UFLIA+H, then there exists a quantifier-free interpolant I in QF_UFLIA+H such that φ ⊨ I and I ∧ ψ is unsatisfiable.

**Why this is hard.** Interpolation in the combined theory is not a free lunch from the component theories. The Nelson-Oppen combination framework gives interpolation for *convex, stably infinite* theories, but the heap theory H is neither convex (disjunctive spatial assertions) nor stably infinite (finite models). We must develop a new combination result that exploits the bounded-depth restriction to recover a form of compactness, then construct interpolants by stitching together arithmetic interpolants with spatial interpolants using a novel *heap projection* operation.

**Proof technique.** Proof-theoretic: define a sequent calculus for QF_UFLIA+H, prove cut-elimination, extract interpolants from cut-free proofs. The cut-elimination argument requires a new commutation lemma for spatial and arithmetic cuts.

### Theorem 2: Decidability and Complexity of Refinement Subtyping with Width Subtyping and Dynamic Keys
**Statement.** The subtyping judgment Γ ⊢ {x: τ₁ | φ₁} <: {x: τ₂ | φ₂} in the presence of width subtyping on structural object types (where τ may be `{k₁: τ₁, ..., kₙ: τₙ, ...}` with a row variable) and predicates mentioning `hasattr(x, k)` for string-valued k is decidable when predicates are drawn from QF_UFLIA+H, and is **coNP-complete**.

**Why this is hard.** Width subtyping introduces an implicit universal quantifier ("for all keys k present in the supertype, ..."), and the interaction between this quantifier and arithmetic refinements on the values associated with keys is subtle. We must show that the universal quantifier can be *Skolemized away* when the set of possible keys is drawn from a finite set extractable from the program text — a non-trivial static analysis result in itself for dynamic languages.

### Theorem 3: Soundness and Completeness of Incremental Fixed-Point Maintenance under Stratified Negation
**Statement.** Let P be a stratified Datalog¬ program encoding the whole-program refinement type inference, and let P' = P[Δ] be the program obtained by replacing clauses corresponding to changed function bodies. The incremental maintenance algorithm (semi-naive delta propagation with stratum-respecting invalidation) computes exactly the minimal model of P' in time O(|Δ_out|·poly(|P|)), where |Δ_out| is the size of the symmetric difference between the old and new minimal models.

**Why this is hard.** Standard incremental Datalog maintenance (DRed, counting algorithms) does not handle stratified negation correctly when the stratum structure is perturbed by the update. We must prove that function-level updates preserve the stratification (a structural lemma about the clause dependency graph), and that the invalidation phase correctly handles derived facts that depend on negated subgoals whose truth value changed.

### Algorithm: CEGAR with Heap-Aware Predicate Discovery
A new CEGAR loop that, given a spurious counterexample trace through the abstract interpreter, extracts a *heap-aware interpolant* (via Theorem 1) and projects it into the predicate abstraction lattice. We must prove **convergence**: the sequence of predicate abstractions is a strictly ascending chain in a lattice of finite height (the height is bounded by the number of program variables × heap depth bound × the number of atomic predicates in QF_UFLIA+H over the program's constants).

## Why Existing Mathematics Is Insufficient

1. **Liquid Types (Rondon, Kawaguchi, Jhala 2008; Vazou et al. 2014)** assume an ML-like type system with no mutation, no heap, no structural subtyping with row polymorphism. Their Horn-clause constraint generation produces a system solvable by predicate abstraction over QF_UFLIA alone. Adding heap predicates breaks the decidability argument entirely.

2. **Separation logic interpolation (Jovanović & Dutertre 2014; Brotherston et al. 2014)** exists for the *pure* heap fragment but has never been combined with arithmetic in a way that preserves quantifier-free interpolation. The combination is the gap.

3. **Incremental Datalog (Gupta et al. 1993; Motik et al. 2019)** handles semi-positive or stratified programs, but the specific clause structure arising from refinement type inference — where negation encodes "this refinement does NOT hold at a particular program point" — has not been analyzed for stratification-preserving updates. The existing theorems do not apply without our structural lemma.

4. **Gradual typing (Siek & Taha 2006) and occurrence typing (Tobin-Hochstadt & Felleisen 2008)** solve a different problem: they assign types to programs that mix typed and untyped code, but they do not produce refinement-level guarantees (bounds safety, division-by-zero freedom). The mathematical gap is the refinement layer.

5. **Abstract interpretation for dynamic languages (Monat et al. 2020; Fromherz et al. 2023)** provides sound value analyses but does not produce *checkable contracts* in the form of refinement types. Bridging the two requires the new subtyping decidability result (Theorem 2) to guarantee that the inferred types are actually checkable.

## Best-Paper Argument (Mathematical Contribution Angle)

The strongest best-paper argument rests on Theorem 1 (interpolation for QF_UFLIA+H). This is a contribution to mathematical logic and SMT theory that stands entirely independently of the application to dynamic languages. Quantifier-free interpolation results for combined theories are rare and technically deep — each new one requires overcoming specific obstacles in the combination. The bounded-heap fragment we define is natural (it corresponds exactly to the heap footprints that arise in frame-bounded abstract interpretation), and the interpolation result has immediate applications beyond type inference: to software model checking (CEGAR for heap-manipulating programs), to invariant inference, and to abductive reasoning in separation logic. A best-paper committee can point to this theorem and say "this is a new result in logic that happens to enable a practical system," which is the strongest possible positioning.

The secondary argument is the novel connection between three previously disjoint bodies of work — Liquid Types, separation logic, and incremental Datalog — unified through a single formal framework. The system is not a bag of heuristics; it is a stack of theorems, each load-bearing, each falsifiable.

## Fatal Flaws

1. **The bounded-heap restriction may be too restrictive.** Real Python programs use recursive data structures (trees, linked lists of arbitrary depth). Bounding the heap depth to a constant *k* means the analysis cannot verify properties of recursive traversals. The counterargument is that most array-bounds and null-safety properties do not require deep heap reasoning, but this limits the "killer app" framing. A reviewer will ask: "what fraction of real bugs require heap depth > k?"

2. **Practical SMT performance is uncertain.** Even if QF_UFLIA+H is decidable, the coNP-completeness of subtyping means that the SMT solver may time out on real-sized programs. The theoretical contribution is clean, but the empirical evaluation may show that the system is too slow for CI on 165K LoC codebases. The incrementality helps, but each individual query may still be expensive.

3. **Predicate language expressiveness vs. decidability tension.** The predicates we can express in QF_UFLIA+H may not capture the invariants programmers actually need. For instance, predicates involving string operations (common in Python/TypeScript) fall outside the fragment. Extending the theory to handle strings pushes decidability into undecidable territory (QF_SLIA is decidable, but combining it with the heap theory may not be). This is an honest tension that the paper must acknowledge rather than hide.

4. **The stratification-preservation lemma (Theorem 3) may fail for higher-order functions.** If a code change alters the call graph (e.g., passing a different callback), the stratum structure of the Datalog encoding may change, invalidating the incremental maintenance guarantee. The paper would need to either prove that call-graph changes preserve stratification (unlikely in general) or fall back to from-scratch reanalysis for such changes (practically acceptable but theoretically unsatisfying).
