# Review: CausalBound — Verified Worst-Case Systemic Risk Bounds via Decomposed Causal Polytope Inference with Adversarial Search

**Reviewer:** Lila Zhang (Symbolic Reasoning & SAT/SMT Solving)
**Score: 7/10**

---

The symbolic reasoning architecture in CausalBound is ambitious and well-conceived at the design level. The streaming SMT co-routine's use of incremental push/pop semantics to maintain a growing assertion context during junction-tree message passing is a principled approach to verified probabilistic inference. The choice to encode causal polytope bounds in QF_LRA is correct—these are fundamentally linear arithmetic claims about the feasible region of the causal polytope LP—and the incremental protocol exploits Z3's and CVC5's native support for assertion stack management. The soundness theorem (New Mathematics #4) requiring that the assertion sequence is monotone—such that SAT for all prefix contexts implies global inference correctness—is the right formalization. This is structurally analogous to the monotonicity requirement in incremental SAT solving (IPASIR interface), and the junction-tree's message-passing schedule provides a natural topological ordering that can ground the monotonicity proof.

My primary concern is with the encoding complexity and its impact on solver performance. The problem statement describes encoding junction-tree messages as QF_LRA assertions, but each message in Lauritzen-Spiegelhalter inference is a marginal potential function—a table of real values indexed by the variables in the separator set. Faithfully encoding a single message as QF_LRA assertions requires one real variable per table entry (up to k^s entries for separator size s and cardinality k) plus normalization and non-negativity constraints. For the claimed parameters (k = 4–6, separator sizes s = 5–15), a single message could generate 4^15 ≈ 1 billion real variables in the worst case—clearly intractable. The system must therefore encode messages at a higher level of abstraction, perhaps using symbolic representations of structured CPDs (e.g., exploiting the piecewise-linear structure of CDS payoffs to encode messages as parameterized linear functions rather than explicit tables). But this higher-level encoding introduces a gap between what the SMT solver verifies and what the inference engine computes: the solver checks the symbolic abstraction, not the concrete numerical computation. This gap must be formally bounded, and the problem statement does not address it. Without a formal abstraction-refinement connection between the symbolic SMT encoding and the concrete SIMD-vectorized numerical computation, the proof certificates verify an abstraction of the inference, not the inference itself—a subtle but critical distinction in any formally verified system.

The integration of graph-theoretic predicates with QF_LRA is the most technically challenging aspect of the symbolic reasoning pipeline, and the problem statement's treatment of it is insufficient. D-separation queries, moral graph construction, and treewidth verification are all graph-theoretic properties that do not reduce naturally to linear arithmetic. The standard approach in SMT is to implement a custom theory solver (à la DPLL(T)) for the graph-theoretic fragment, communicating with the QF_LRA solver via Nelson-Oppen theory combination. This is feasible but non-trivial: the graph-theory solver must support incremental assertion, backtracking, and theory propagation. An alternative is to encode graph properties as propositional variables (e.g., Boolean edge indicators with transitivity axioms for reachability), but this can cause a combinatorial blowup. The 7,200 algorithmic LoC budget for the streaming SMT co-routine must accommodate either approach, and both require substantial implementation effort beyond the LP bound encoding. I would recommend the authors clarify whether they plan a custom DPLL(T) theory solver or a propositional encoding, and provide a complexity analysis of either approach for graphs of the target size (50–500 nodes). The claim that "the causal structure makes verification tractable (d-separation reduces the assertion space)" is appealing but requires quantification: how many assertions are eliminated by d-separation pruning in practice, and does this reduction suffice to bring the total assertion count within the solver's performance envelope?
