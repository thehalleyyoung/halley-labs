# Bounded-Rational Usability Regression Testing: Reducing Cognitive-Model Composition to Constrained Inference over UI State Spaces

## Problem and Approach

Usability evaluation remains one of the last manual bottlenecks in software engineering. Despite decades of work on cognitive architectures (ACT-R, EPIC) and automated usability tools (CogTool, guideline-based checkers), no existing system closes the loop from raw UI structure to cognitively-grounded regression verdicts within the time and compute budgets of continuous integration. The gap is not merely engineering: it is mathematical. Composing heterogeneous cognitive laws—Fitts' motor targeting, Hick–Hyman choice complexity, visual-search saliency, working-memory decay—into a single coherent cost model over sequential UI interactions requires a formal framework that existing tool pipelines lack. We argue that information-theoretic bounded rationality, specifically the free-energy formulation of Ortega & Braun (2013), supplies exactly the missing unifier: a single variational objective $F(\pi) = \mathbb{E}_\pi[R] - \frac{1}{\beta} D_{\mathrm{KL}}(\pi \| p_0)$ that trades task reward against information-processing cost, recovering Fitts', Hick–Hyman, and visual-search phenomena as capacity-constrained channels under per-subsystem capacity parameters $\beta_i$ sharing a common variational form (cf. MacKenzie 1992 for Fitts-as-capacity; Wickens 2002 for multiple-resource theory).

Our central technical claim is that usability regression testing—detecting when a UI change degrades the cognitive cost of completing a task—can be reduced to constrained inference over bounded-rational cognitive models composed from UI accessibility trees. We do *not* claim absolute prediction of human performance (a fidelity oracle); we claim *relative consistency* (a consistency oracle): given two UI versions and a task specification, the system determines whether the cognitive cost distribution has shifted adversely, with formal error bounds. This scoping is critical. Absolute prediction requires ecological validity at a level no computational cognitive model has achieved in the wild. Regression detection requires only that the model's *ordering* of costs is preserved under perturbation—a weaker condition whose empirical validation (rank-correlation between model orderings and human preference orderings on held-out UI pairs) is tractable via retrospective comparison against existing published human-performance data (see Evaluation Plan), analogous to how compiler optimizations need not predict wall-clock time but must preserve asymptotic complexity orderings.

The reduction proceeds in three stages. First, the system parses a live UI accessibility tree into a task-state Markov Decision Process (MDP) whose states encode perceptual groupings, interactive affordances, and working-memory load. Second, a compositional cognitive cost algebra annotates each transition with information-theoretic cost terms derived from calibrated cognitive laws, yielding a weighted MDP. Third, bounded-rational policies (softmax over free energy) generate trajectory distributions whose statistics—expected cost, variance, tail risk, bottleneck locations—are compared across UI versions via hypothesis tests with controlled false-discovery rates. When a regression is detected, an information-theoretic bottleneck taxonomy classifies the failure mode (perceptual overload, choice paralysis, motor difficulty, memory decay, cross-channel interference), providing actionable diagnostics to developers. As a stretch goal, an SMT-backed repair synthesizer proposes minimal UI mutations that restore the cost envelope; this extension builds on the core pipeline but is not required for the system's primary value.

The entire pipeline is designed for laptop-CPU execution within CI/CD wall-clock constraints: no GPU, no cloud dependency, no human participants. This is not a concession but a design principle. Saliency computation derives from structural features of the accessibility tree (semantic roles, spatial layout, label informativeness) rather than pixel-level vision models. Monte Carlo trajectory sampling is embarrassingly parallel across CPU cores. SMT and ILP solvers used for repair synthesis are CPU-native. And the bounded-rational bisimulation reduction—our first theoretical contribution—ensures that abstract MDPs remain small (empirically ≤10K states for production UIs), keeping all inference tractable.

We position this work at the intersection of computational interaction (Oulasvirta et al.), information-theoretic decision-making (Ortega & Braun 2013, Tishby et al. 2000), and automated program analysis (where the consistency-oracle framing draws direct analogy to regression test oracles, type-checkers, and static analyzers—tools that are useful precisely because they are conservative approximations, not omniscient predictors). The system is, to our knowledge, the first to automatically construct task-level cognitive simulations from UI accessibility trees and use information-theoretic bottleneck analysis to produce cognitively-grounded regression verdicts within CI/CD constraints. We scope this claim carefully: CogTool (John & Salvucci 2005) pioneered cognitive modeling for UI evaluation but required manual task specification and did not target regression detection; Ivory & Hearst (2001) surveyed automated usability evaluation but predated information-theoretic and bounded-rationality frameworks; St. Amant et al. (2004) explored simulated interaction but not compositional cost algebras or bottleneck taxonomies. Our contribution is the formal reduction and its end-to-end realization.

## Value Proposition

**Who needs this.** Every software team shipping user-facing products performs usability evaluation, and almost all do it manually: recruiting participants, running studies, interpreting results over days or weeks. Meanwhile, code ships hourly. The result is that usability regressions escape into production at rates that dwarf functional regressions, because there is no automated oracle analogous to a unit test suite. Design-system teams at scale (enterprise SaaS, mobile platforms, accessibility-critical domains like healthcare and government) are the acute audience: they maintain hundreds of components, each change potentially degrading task flows that span multiple components, and they have no way to detect this before users complain.

**What becomes possible.** With a bounded-rational usability oracle integrated into CI/CD, every pull request that modifies UI structure receives an automatic cognitive cost diff: a report indicating whether any task's expected cost, variance, or tail risk has increased beyond a configurable threshold, classified by bottleneck type. This transforms usability from a periodic qualitative assessment into a continuous quantitative signal—not replacing human studies, but triaging and prioritizing them. Teams that currently run usability studies quarterly could restrict those studies to the genuinely novel questions the oracle cannot answer, while catching regressions in hours rather than months.

**Why not LLMs?** Large language models can provide broad, qualitative usability critique and are increasingly used for heuristic evaluation. However, CI/CD regression detection imposes four requirements that LLMs fundamentally cannot satisfy. First, *determinism*: the same UI pair must produce the same verdict across runs, enabling developers to trust that a passing gate remains passing. LLM outputs are stochastic and sensitive to prompt variation. Second, *quantitative comparability*: the system must produce a scalar or structured cost differential that can be thresholded, trended over time, and compared across releases. LLM outputs are natural language, requiring further interpretation. Third, *monotonicity*: if UI version B is strictly worse than A on all task dimensions, the system must never report an improvement. LLMs provide no such guarantee. Fourth, *formal error bounds*: the system must characterize its approximation error so that teams can calibrate thresholds to their risk tolerance. LLMs offer no error theory. We position this system as complementary to LLM-based usability critique: LLMs provide broad qualitative feedback during design exploration, while the bounded-rational oracle serves as a quantitative CI gate with provable properties.

## Scope and Limitations

This system detects **structural usability regressions** visible in accessibility tree changes—regressions in information architecture, interaction flows, element counts, groupings, navigation depth, and affordance structure. It does **not** detect visual usability regressions: changes in CSS styling, typography, color contrast, spacing, animation timing, or layout aesthetics. This scoping is deliberate, not a limitation to be overcome. Structural regressions are the class most affected by code changes in pull requests (component refactors, feature additions, navigation restructuring) and least served by existing automated tools. Visual regressions are well-addressed by screenshot-diff tools (Chromatic, Percy, BackstopJS) that compare rendered pixel output. The two approaches are complementary: screenshot-diff tools catch visual regressions that leave the accessibility tree unchanged, while this system catches structural regressions that leave screenshots superficially similar but degrade task performance. A mature CI/CD pipeline would employ both.

The system further requires that UIs expose a well-formed accessibility tree (via platform accessibility APIs or ARIA-annotated web UIs) and that task specifications be provided declaratively. The accessibility-tree requirement aligns with legal mandates (WCAG, Section 508, EN 301 549) that increasingly compel well-structured accessibility annotations. The task-specification requirement can be partially automated for common patterns (form completion, navigation, search) but remains a setup cost for novel task flows.

## Technical Difficulty

The problem decomposes into three hard subproblems, each requiring genuine theoretical work beyond engineering integration.

**Subproblem 1: Bounded-rational state abstraction.** Production UIs generate MDPs with $10^4$–$10^6$ raw states (every combination of visible elements, focus positions, scroll offsets, modal layers). Exact inference is intractable. We require a principled abstraction that preserves the *behaviorally relevant* structure under bounded-rational policies—not arbitrary bisimulation, but bisimulation calibrated to the information-processing capacity $\beta$ of the cognitive model. This yields the bounded-rational bisimulation theory (Contribution 1), which is mathematically novel: classical bisimulation ignores agent capacity; our metric $d_{\mathrm{cog}}$ weights state distinguishability by the agent's ability to *exploit* that distinction.

**Subproblem 2: Compositional cost modeling.** Cognitive laws interact nonlinearly. Fitts' motor time depends on the visual search time that preceded target acquisition; Hick's choice time depends on working-memory load from prior steps; interference between parallel perceptual channels (cf. Wickens 2002, Multiple Resource Theory) modulates both. We need a compositional algebra that (a) has clean formal semantics, (b) yields tractable computation, and (c) upper-bounds the true mutual information of the full discrete-event cognitive simulation. This yields Contribution 2: the cognitive cost algebra with operators $\oplus$ (sequential composition), $\otimes$ (parallel interference), and $\Delta$ (context modulation), with a soundness theorem bounding divergence from ground-truth simulation.

**Subproblem 3: Bottleneck classification and repair synthesis.** Detecting that a regression occurred is insufficient; the system must explain *why* to guide developers. The information-theoretic bottleneck taxonomy (Contribution 3) classifies failure modes by their channel-capacity signatures, enabling targeted diagnostics. As a stretch goal, repair synthesis via SMT/ILP constraint solving over the UI's structural parameters can propose concrete fixes, but the core system delivers value through detection and classification alone.

## Incremental Realization

The theoretical contributions build incrementally on a minimal viable system, ensuring that each layer of sophistication adds measurable value to a functioning baseline.

**Minimal viable system.** The simplest instantiation of the pipeline requires three components: (1) an accessibility-tree parser that extracts interactive elements, their semantic roles, and spatial layout; (2) an additive cost model using classical Fitts' law ($\text{MT} = a + b \cdot \log_2(1 + D/W)$) for motor operations and Hick–Hyman law ($\text{RT} = a + b \cdot \log_2(n)$) for choice operations, summed over task steps with no interaction terms; and (3) a scalar regression threshold that flags a pull request when the total cost difference exceeds a configurable percentage. This MVP requires no bisimulation, no cost algebra, no bottleneck taxonomy—and no novel composition parameters. It can be implemented, tested, and validated against published task-completion data as a standalone tool.

**Contribution 1 (bisimulation) enables scaling.** The MVP's additive cost model operates on the raw accessibility tree, which suffices for small UIs (≤200 elements) but becomes intractable for large applications. Bounded-rational bisimulation provides principled state-space reduction that preserves cost orderings, extending the system to production-scale UIs with thousands of interactive elements while maintaining the formal guarantee that if a regression exists in the full MDP, it is detected in the abstract MDP.

**Contribution 2 (cost algebra) captures nonlinear interactions.** The MVP's additive model ignores the reality that cognitive operations interact: prior memory load amplifies subsequent choice time, parallel perceptual demands interfere with motor execution. The compositional cost algebra replaces simple summation with operators that model sequential coupling ($\oplus$), parallel interference ($\otimes$), and context modulation ($\Delta$), yielding more accurate cost estimates and fewer false negatives on regressions caused by interaction effects.

**Contribution 3 (taxonomy) provides actionable diagnostics.** The MVP reports only that a regression occurred and its magnitude. The bottleneck taxonomy classifies *why*—perceptual overload, choice paralysis, motor difficulty, memory decay, or cross-channel interference—enabling developers to target fixes without manual usability analysis. Each classification maps to a specific repair strategy, transforming the system from a regression detector into a diagnostic tool.

**Stretch goal: repair synthesis.** Beyond classification, SMT-backed synthesis can propose concrete UI mutations that restore the cost envelope. This extension leverages the bottleneck taxonomy's intervention-specificity property but requires additional engineering (constraint encoding, solver integration, repair validation) that is not essential to the core value proposition.

## New Mathematics Required

### Contribution 1: Bounded-Rational Bisimulation

**Definition.** Given an MDP $(S, A, T, R)$ and a bounded-rational policy family $\pi_\beta(a|s) \propto \exp(\beta \cdot Q(s,a))$, define the cognitive distance metric:

$$d_{\mathrm{cog}}(s_1, s_2) = \sup_{\beta' \leq \beta} \, d_{\mathrm{TV}}\!\bigl(\pi_{\beta'}(\cdot|s_1),\, \pi_{\beta'}(\cdot|s_2)\bigr)$$

An abstraction $\varphi: S \to \hat{S}$ is a **bounded-rational $\varepsilon$-bisimulation** if $\varphi(s_1) = \varphi(s_2) \implies d_{\mathrm{cog}}(s_1, s_2) \leq \varepsilon$.

**Main Theorem (Trajectory Approximation).** If $\varphi$ is a bounded-rational $\varepsilon$-bisimulation, then for any task of horizon $H$, the total variation distance between trajectory distributions in the original and abstract MDPs satisfies $d_{\mathrm{TV}}(\mathcal{T}, \hat{\mathcal{T}}) \leq 1 - (1 - \beta\varepsilon)^H$. For typical parameters ($\beta=5, \varepsilon=0.005, H=30$), this yields $d_{\mathrm{TV}} \leq 0.53$. This is a worst-case bound on *absolute* trajectory approximation—it guarantees that any single trajectory statistic computed under the abstract MDP is within 0.53 TV of the original, which is loose for general purposes. However, usability regression detection does not require tight absolute approximation; it requires accurate *comparison* between two UI versions. The following theorem shows that the regression-detection task enjoys a much tighter bound.

**Paired-Comparison Theorem.** When two UI versions $A$ and $B$ are both abstracted using the *same* bisimulation partition $\varphi$ (i.e., both accessibility trees are coarsened under a shared partition refined from their union), the regression detection error for cost differences satisfies:

$$|(\hat{C}_B - \hat{C}_A) - (C_B - C_A)| \leq O(\varepsilon)$$

rather than the $O(H\beta\varepsilon)$ bound implied by the absolute trajectory approximation theorem applied independently to each version. The key insight is that systematic abstraction errors—the bias introduced by merging states within each $\varepsilon$-ball—are correlated across $A$ and $B$ when both are abstracted under the same partition. In the cost difference $\hat{C}_B - \hat{C}_A$, these correlated errors cancel to first order, leaving only a residual proportional to the partition granularity $\varepsilon$ rather than the horizon-amplified $H\beta\varepsilon$. This is the formal basis for the consistency-oracle claim: while absolute cost predictions may be loose, cost *orderings* between UI versions are preserved with high fidelity under shared abstraction.

**Coarsening Corollary.** The abstract MDP size scales as $|\hat{S}| = O(\beta^d \cdot N_\varepsilon(\mathcal{F}))$ where $d$ is the metric doubling dimension of $(S, d_{\mathrm{cog}})$ and $N_\varepsilon(\mathcal{F})$ is the $\varepsilon$-covering number of the perceptual feature space under $\ell_\infty$. Empirically, for production UIs with 200–2000 interactive elements, $|\hat{S}| \leq 10^4$ (validated on accessibility-tree snapshots from open-source applications).

### Contribution 2: Compositional Cognitive Cost Algebra

**Definition.** A cost element $c = (\mu, \sigma^2, \kappa, \lambda)$ encodes the mean, variance, capacity utilization, and interference susceptibility of a cognitive operation. Three operators:

- $c_1 \oplus c_2$ (sequential): $\mu_{1+2} = \mu_1 + \mu_2 + \gamma \cdot \kappa_1 \cdot \mu_2$ (prior load amplifies subsequent cost)
- $c_1 \otimes c_2$ (parallel interference): $\mu_{1 \times 2} = \max(\mu_1, \mu_2) + \alpha \cdot \lambda_1 \lambda_2 \cdot \min(\mu_1, \mu_2)$
- $\Delta_\theta(c)$ (context modulation): scales $(\mu, \sigma^2)$ by context parameters $\theta$ encoding memory load, fatigue, and task urgency.

**Soundness Theorem.** For any task graph $G$ with $n$ operations, the composed cost $C_{\mathrm{alg}}(G)$ satisfies:

$$C_{\mathrm{alg}}(G) \geq \sum_{t=1}^{n} I(S_t; A_t)$$

where $I(S_t; A_t)$ is the mutual information between state and action at step $t$ in the full discrete-event cognitive simulation. The algebra thus provides a conservative upper bound on true information cost—the useful direction for regression detection, since any cost increase in the algebra implies a genuine increase in cognitive information processing. For tree-structured task graphs with bounded interference degree $k$, a tighter relative-error bound holds: $C_{\mathrm{alg}}(G) \leq (1 + O(k\varepsilon/\beta)) \sum_t I(S_t; A_t)$.

*Proof sketch and required assumptions.* The soundness theorem holds for discrete-event cognitive simulations satisfying three conditions: (i) Markov transitions—the state at step $t+1$ depends on the state and action at step $t$ but not on the full history; (ii) conditionally independent sensory channels—given the world state, perceptual observations across modalities (visual, auditory, proprioceptive) are independent, consistent with the Multiple Resource Theory assumption (Wickens 2002); and (iii) bounded interference degree $k$—each cognitive operation directly interferes with at most $k$ other concurrent operations. Under these assumptions, the proof proceeds by induction on the task graph structure, applying the data-processing inequality at each composition step. For a single operation, the base case holds because each cognitive channel is a noisy channel from state to action, and $I(S_t; A_t)$ is a lower bound on the processing cost by the rate-distortion principle. For sequential composition ($\oplus$), the coupling term $\gamma \cdot \kappa_1 \cdot \mu_2$ upper-bounds the conditional mutual information $I(S_{t+1}; A_{t+1} | S_t, A_t)$ that arises from load propagation: the data-processing inequality applied to the cascade $S_t \to A_t \to S_{t+1} \to A_{t+1}$ shows that upstream capacity utilization $\kappa_1$ gates the additional information that must be processed downstream. For parallel composition ($\otimes$), the interference term $\alpha \cdot \lambda_1 \lambda_2 \cdot \min(\mu_1, \mu_2)$ bounds the mutual information $I(A_t^{(1)}; A_t^{(2)} | S_t)$ between nominally independent action channels sharing a common cognitive resource, again via the data-processing inequality applied to the shared-resource bottleneck. For context modulation ($\Delta_\theta$), the scaling preserves the bound by monotonicity of mutual information under channel degradation. The tree-structure and bounded-degree conditions ensure that these per-step bounds compose without combinatorial blowup, yielding the global soundness guarantee.

**Parameter calibration.** The composition parameters $\gamma$ (sequential coupling) and $\alpha$ (parallel interference) are novel to this algebra—unlike the component-level parameters (Fitts' $a, b$; Hick–Hyman $a, b$; working-memory decay rate), which have extensive published psychophysical estimates, $\gamma$ and $\alpha$ lack direct published values. We address this gap through three complementary strategies. First, sensitivity analysis: we systematically vary $\gamma$ and $\alpha$ across wide ranges ($\gamma \in [0, 0.5]$, $\alpha \in [0, 1]$) and verify that the *ordinal ranking* of UI versions is robust—that is, the system's regression verdicts are stable across parameter settings, even when absolute cost magnitudes shift. Second, calibration against existing published task-completion data: we fit $\gamma$ and $\alpha$ to minimize rank-order disagreement between model predictions and published multi-step task-completion times from the CogTool validation corpus and Oulasvirta group datasets. Third, we note that the minimal viable system (additive composition) sidesteps these parameters entirely by setting $\gamma = \alpha = 0$, providing a parameter-free baseline against which the full algebra's incremental benefit can be measured.

### Contribution 3: Information-Theoretic Bottleneck Taxonomy

**Definition.** At each decision point in the task MDP, model the cognitive operation as a noisy channel $S_t \to A_t$. Define five bottleneck types by their information-theoretic signatures:

| Type | Signature | Interpretation |
|------|-----------|----------------|
| Perceptual | $H(S_t \| \text{display}) > \tau_p$ | Display entropy exceeds perceptual channel capacity |
| Choice | $\log|A_t| - I(S_t; A_t) > \tau_c$ | Action space entropy far exceeds transmitted information |
| Motor | $I(A_t; \text{target}) < \tau_m$ given $I(S_t; A_t) \geq \tau_c$ | Information reaches decision but not motor execution |
| Memory | $I(S_t; S_{t-k}) < \tau_\mu$ for required $k$ | Temporal information transmission decays below threshold |
| Interference | $I(A_t^{(1)}; A_t^{(2)} \| S_t) > \tau_\iota$ | Spurious dependence between nominally independent channels |

**Exhaustiveness Theorem.** Any decision point whose total cognitive cost exceeds the free-energy baseline by more than $\delta$ must exhibit at least one of the five bottleneck signatures with threshold exceedance $\geq f(\delta, \beta)$.

**Distinguishability Theorem.** For generic UI configurations (outside a measure-zero degenerate set in the parameter space), the five bottleneck signatures are locally separable: each signature's gradient with respect to UI parameters points in a linearly independent direction, enabling dominant-bottleneck identification. In non-generic configurations, multiple bottleneck types may co-occur, but the taxonomy still provides a soft classification with well-defined posterior weights over bottleneck types.

**Intervention-Specificity Theorem.** Each bottleneck type maps to a distinct class of UI repairs (perceptual → grouping/contrast, choice → progressive disclosure, motor → target sizing/placement, memory → persistent state cues, interference → modality separation), and the mapped repair class reduces the bottleneck's signature while leaving others within $O(\varepsilon)$. This theorem provides the formal foundation for repair synthesis: given a classified bottleneck, the corresponding repair class defines the search space for concrete UI mutations. While the full SMT-backed repair synthesizer is a stretch goal, the intervention-specificity property is independently valuable as a guide for manual developer action.

## Best Paper Argument

A best-paper committee rewards work that opens a new research direction, demonstrates surprising technical depth, and offers broad practical impact. This submission delivers on all three criteria.

**New direction.** It establishes usability regression testing as a formal inference problem, analogous to how type theory formalized program correctness. The bounded-rational bisimulation and compositional cost algebra are general tools that extend beyond usability to any domain requiring compositional modeling of capacity-limited agents (autonomous driving interfaces, medical device interaction, accessibility compliance). The consistency-oracle framing resolves a decades-old tension in automated usability evaluation—the gap between model fidelity and practical utility—by demonstrating that regression detection requires strictly weaker assumptions than absolute prediction.

**Technical depth.** The three contributions span MDP theory, information theory, and compositional semantics, connected by a single variational principle. The bounded-rational bisimulation metric is, to our knowledge, novel: existing bisimulation literature assumes either fully rational or fully stochastic agents, not the intermediate regime that characterizes human cognition. The paired-comparison theorem transforms a seemingly loose approximation guarantee into a tight result for the specific task of regression detection—demonstrating that the consistency-oracle framing is not merely a rhetorical device but has concrete mathematical consequences. The soundness theorem for the cost algebra provides, to our knowledge, the first formal guarantee linking a compositional *information-theoretic* cost heuristic to discrete-event cognitive simulation ground truth.

**Practical impact.** The system runs on commodity hardware in CI/CD pipelines, requires no human participants for regression detection, and produces actionable bottleneck diagnostics. It does not replace human usability studies but *complements* them—precisely the positioning that maximizes adoption. The analogy to compilers, profilers, and static analyzers is apt: no one argues that GCC's optimizer "replaces" algorithm design, yet no one ships without it.

## Evaluation Plan

All evaluation is fully automated; no human subjects are required for the regression-detection claim.

**Benchmark suite.** Curate 200+ UI version pairs from open-source projects (e.g., Material UI, Ant Design changelogs) with known usability-affecting changes. Label ground truth via (a) issue-tracker annotations where designers explicitly flagged regressions (acknowledging that this source is biased toward severe regressions and systematically misses subtle degradations), and (b) synthetic mutations (inject known bottleneck-inducing modifications into clean UIs, providing controlled coverage of subtle regression types).

### Retrospective ordinal validation

The system's core claim is that its cost orderings are consistent with human-perceived usability orderings. We validate this claim against existing published human performance data, requiring no new human studies. Specifically, we compute model cost orderings on UI pairs for which human preference or performance orderings are already available in the literature: (1) CogTool validation datasets, which include human task-completion times for multiple interface variants across a range of desktop and mobile applications; (2) interaction log datasets from the Oulasvirta group (Aalto University), which provide task-completion times and error rates for structured interaction tasks; and (3) published UI comparison studies reporting human preference orderings or measured performance differences between interface versions (e.g., Findlater & McGrenere 2004, Gajos et al. 2010, Cockburn et al. 2007). For each dataset, we extract or reconstruct the accessibility-tree representations of the UI versions, run the full pipeline (parse → abstract → cost → compare), and compute rank correlation (Kendall's $\tau$, Spearman's $\rho$) between model-predicted cost orderings and human-measured orderings. We set a minimum threshold of $\tau \geq 0.6$ for ordinal agreement, consistent with the correlation levels achieved by validated cognitive models (e.g., CogTool reports $r \geq 0.8$ for absolute time prediction on in-domain tasks; our weaker ordinal claim should achieve at least moderate rank correlation). This validation is part of the core evaluation, not follow-up work: if the system's orderings do not correlate with human orderings on published data, the consistency-oracle claim fails regardless of internal metrics.

### Regression detection metrics

Precision, recall, and F1 of the oracle's regression verdicts against ground-truth labels. Target: F1 ≥ 0.80 on the curated benchmark. False-positive rate on known-benign refactors (UI changes that do not affect task flows): target ≤ 5%.

### Bottleneck classification metrics

On synthetic mutations with known bottleneck type, measure classification accuracy. Target: ≥ 85% top-1 accuracy across the five-type taxonomy.

### Repair synthesis metrics (stretch targets)

For each detected regression, measure whether the synthesized repair (a) eliminates the flagged bottleneck (cost returns to pre-regression envelope) and (b) does not introduce new bottlenecks. Target: ≥ 70% valid repairs. These metrics are stretch targets contingent on completing the SMT-backed repair synthesizer; the core system is evaluated on detection and classification alone.

### Ablation studies

(1) Bisimulation vs. no abstraction: measure speedup and accuracy preservation. (2) Full cost algebra vs. additive-only (no $\otimes$, no $\Delta$): measure regression-detection accuracy drop, isolating the contribution of composition parameters $\gamma$ and $\alpha$. (3) Bottleneck taxonomy vs. scalar-only cost reporting: measure developer-actionability via repair success rate. (4) Sensitivity analysis over composition parameters: vary $\gamma \in [0, 0.5]$ and $\alpha \in [0, 1]$ and measure ordinal stability of regression verdicts across the parameter range.

### Baselines

(1) Axe-core / Pa11y accessibility checkers (rule-based, no cognitive model). (2) CogTool-style KLM predictors (cognitive but not compositional, not information-theoretic). (3) Random-mutation baseline for repair synthesis. (4) Scalar cost threshold (detect regression if total cost increases by >$x$%) without bottleneck decomposition.

### Performance

Wall-clock time per UI-version-pair analysis on a 4-core laptop CPU. Target: ≤ 60 seconds for UIs with ≤ 500 interactive elements, ≤ 300 seconds for UIs with ≤ 2000 elements.

## Laptop CPU Feasibility

The system is designed from first principles for CPU execution, not adapted from a GPU pipeline.

**Saliency without vision models.** Visual saliency is computed from the accessibility tree's structural properties—semantic roles (button, heading, link), spatial bounding boxes, label token frequencies, ARIA attributes—using closed-form information-theoretic measures (entropy of element-type distributions, mutual information between spatial position and semantic role). No convolutional networks, no pixel rendering.

**Monte Carlo on CPU cores.** Trajectory sampling under softmax policies is embarrassingly parallel: each trajectory is independent, requiring only MDP transition lookups and random-number generation. On a 4-core machine with 8 threads, 10K trajectories over a 50-step horizon complete in seconds for MDPs with ≤10K states.

**SMT/ILP solvers are CPU-native.** Repair synthesis formulates constraints over discrete UI parameters (element positions, groupings, label assignments) and dispatches to Z3 or CBC, both of which are designed for and optimized on CPU architectures.

**Bisimulation keeps MDPs small.** The bounded-rational bisimulation coarsening is the key enabler: by merging states that a capacity-limited agent cannot distinguish, it reduces raw UI state spaces (potentially $10^5$–$10^6$) to abstract MDPs of $\leq 10^4$ states. All subsequent inference—policy computation, trajectory sampling, bottleneck analysis—operates on the abstract MDP.

**No training phase.** The cognitive laws (Fitts', Hick–Hyman, visual-search models, working-memory decay) are calibrated from published psychophysical parameters, not learned from data. The system has no training loop, no gradient computation, no batch processing. It is an inference engine, not a learning system.
